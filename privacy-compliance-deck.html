The mandate

Why this matters now

Modern US privacy law gives people a single switch to say "don't sell or share my data." As a tag that lives on hundreds of client sites, we have to respect it everywhere — without asking each site to do a thing.

The signal
A browser-level opt-out (Global Privacy Control) a user sets once — recognized under CCPA/CPRA, Colorado & Connecticut.
The request
Explicit consumer "Do Not Sell or Share" submissions that must be recorded and suppressed downstream.
Two obligations → two layers. Stop tracking users who signal opt-out, and honor people who formally ask to be removed.
Previous version → current version

What changed in the tag

Three upgrades took the tag from a privacy-blind script to a compliant, defensive one — with zero change for users who never opted out.

Previous Privacy-blind
  • − Initialized for every visitor — no privacy-signal check
  • − A throwing callback could crash the host page
  • − Loose types; assumed each host page's globals
Current Compliant & defensive
  • + Honors GPC / DNT — hard no-track for opt-outs
  • + Error boundary — a failure can't crash the page
  • + Strict types; host globals treated as unknown
The user's choice · current behavior

Two journeys, one rule

The decision is made before any data is collected or any cookie is set — at the very top of the tag.

User is OK with tracking

No opt-out signal present. The tag initializes normally:

  • Analytics & attribution run as designed
  • Events sent, cookies set
  • Behaves exactly as before — zero change

User has opted out

Browser signals GPC or Do Not Track. The tag never starts:

  • Hard no-track — nothing initializes
  • No events, no cookies, no network calls
  • Applies on every site, automatically
Change 01 · Opt-out handling

From "track everyone" to "honor the signal"

Previous Always on
  • − The tag initialized for every visitor
  • − No Global Privacy Control check at all
  • − Do Not Track, at best, only limited what was sent — data was still collected and cookies still set
Current Gated at the top
  • + Reads GPC + DNT and returns before anything runs
  • + Opted-out visitors → zero events, zero cookies
  • + One check in the shared tag → every site inherits it
No change for users who haven't opted out — the new behavior is the opt-out path only.
Change 01 · In code

The opt-out gate, in code

Two small, dependency-free additions — a pure signal detector, and a one-line gate at the very top of the tag.

tracker-core/utils/privacy-opt-out.ts
// New file — detect US opt-out signals.
const isDntEnabled = (value: string | null | undefined): boolean =>
  value === "1" || value === "yes";

export const isUsPrivacyOptOut = (): boolean => {
  const gpcOptOut = navigator.globalPrivacyControl === true;

  // each DNT source checked independently
  const dntOptOut =
    isDntEnabled(navigator.doNotTrack) ||
    isDntEnabled(navigator.msDoNotTrack) ||
    isDntEnabled(window.doNotTrack);

  return gpcOptOut || dntOptOut;
};
apps/tracker/src/index.ts
const context = getContext();

// Gate: honor GPC / DNT before anything runs.
if (isUsPrivacyOptOut()) {
  logger.debug("opt-out detected (GPC/DNT)");
  return;            // hard no-track
}

await getCustomTags();
await getAppIdTags(context.appId);
No new dependencies, no per-site config — the gate sits above adapters, Snowplow, custom tags and app-id tags.

Layer 02 · Server-side opt-out pipeline

Honor explicit "remove me" requests

When a consumer formally opts out — a "Do Not Sell or Share" request under CCPA/CPRA — we capture it through a secured endpoint and process it into a suppression list, plus a record we can audit.

Where
mediajel-gql-service — our core API
How it's secured
Access-controlled endpoint — per-partner opt-out tokens + API key
What's captured
Email, mobile ad ID, IP, name & address — to match and suppress
A queued batch job processes new requests, builds the suppression list, and is idempotent — the same request is never double-counted.

Change 02 · Trust & robustness

From "could crash your site" to "fails quietly"

A third-party script that crashes a client's page is a liability. Being safe and reliable is part of respecting the user.

Previous Fragile guest
  • − A throwing callback (listener / interceptor / timer) could bubble to the host page
  • − Assumed host globals (window, dataLayer, Shopify…) → runtime crashes
  • − Loose typing hid these risks
Current Good guest
  • + Every callback wrapped in an error boundary — logged, contained
  • + Safe DOM access + defensive parsing
  • + Strict TypeScript; external globals optional / unknown
  • + Proven by an E2E "can't crash the page" test
Change 02 · In code

The hardening, in code

tracker-core/utils/guard.ts
// Wrap every callback we register so a throw
// is logged & swallowed — never the host page.
export const guard = <A extends any[], R>(
  fn: (...args: A) => R,
  label = "callback",
) => {
  return function (this: unknown, ...args: A) {
    try {
      return fn.apply(this, args);
    } catch (err) {
      logger.error(`"${label}" threw; suppressed`, err);
      return undefined;
    }
  };
};
tracker-core/interface.ts — honest types
declare global {
  interface Navigator {
    globalPrivacyControl?: boolean;
  }
  interface Window {
    // our injected infra: present
    tracker: SnowplowTrackerFn;
    // external globals: optional → guard
    Shopify?: { checkout?: any };
  }
}
e.g. ecommerce adapter — now guarded
- tracker.ecommerce.trackTransaction(evt);
+ tracker.ecommerce?.trackTransaction(evt);
Strict TypeScript forced every external-global access to be guarded — surfacing latent crashes at compile time, not on a client's page.

The landscape

A patchwork you can't hand-build per site

US privacy law is now state-by-state. A growing number of states require honoring a browser universal opt-out mechanism (UOOM) — so one automatic signal beats a banner on every site.

Laws that recognize a universal opt-out

CaliforniaCCPA / CPRA
ColoradoCPA
ConnecticutCTDPA
TexasTDPSA
OregonOCPA
MontanaMCDPA
+ moreadopting UOOM

The one signal that covers them

Global Privacy Control (GPC) is the industry spec for a universal opt-out — a single browser flag (Sec-GPC header + navigator.globalPrivacyControl) that asserts the user's choice across all sites. Honor it once and the UOOM-mandating states are covered.
